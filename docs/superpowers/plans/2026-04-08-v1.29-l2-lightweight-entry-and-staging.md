# v1.29 L2 Lightweight Entry And Staging Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a low-friction but trust-honest `L2` staging path so AITP can record reusable insight candidates from discussion or literature work before full canonical promotion.

**Architecture:** Introduce a schema-backed staging entry format under `canonical/staging/`, implement staging write/read helpers inside `l2_graph.py`, and expose the path through `AITPService` and CLI. Keep staging explicitly provisional and separate from canonical `L2`, but allow optional retrieval so recent staged insights can start helping later work without pretending they are promoted truth.

**Tech Stack:** Python 3.10+, `pytest`, JSON Schema, AITP CLI/service, existing `canonical/` retrieval code

---

## Scope Guard

This plan implements only the lightweight entry and staging path.

It does **not** implement:

- automatic promotion from staging into canonical `L2`,
- `physical_picture` canonical activation,
- embedding search,
- or full runtime-loop automation over staged memory.
