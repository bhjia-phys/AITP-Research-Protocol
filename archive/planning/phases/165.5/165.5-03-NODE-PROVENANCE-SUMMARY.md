# Phase 165.5-03 Node Provenance Summary

Status: implemented in working tree

## Goal

Close the next `165.5-03 / 999.84` slice by making exported concept-graph node
pages materially easier to read: source anchors should carry provenance
excerpts, and relation text should read as directional statements instead of
bare symbolic rows.

## What Landed

- `obsidian_graph_export.py` now accepts `source_rows` and builds a
  source-level lookup for exported node pages
- source-anchor sections in exported node notes now include, when available:
  - source summary excerpt
  - best locator text (`abs_url`, absolute path, source URL, or local path)
- relation wording is now directional:
  - outgoing edges: `This node <relation> <target>`
  - incoming edges: `<source> <relation> this node`
  - hyperedge context: `This node participates in <relation> with ...`
- `l1_vault_support.py` now passes topic `source_rows` into the graph export,
  so provenance rendering is driven by the same topic-scoped source manifest
  already used elsewhere in the `L1` vault

## Outcome

`165.5-03` now has another closed readability slice:

- exported graph notes show why a node matters without forcing the operator to
  jump back to raw source indexes
- source provenance is visible directly on the node page
- relation sections read like navigable graph statements instead of raw edge
  tuples
