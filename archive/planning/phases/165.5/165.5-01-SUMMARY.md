# Phase 165.5-01 Summary

Status: materially implemented in working tree

## Goal

Start the `165.5` L0/L1 integration phase by extending the existing
arXiv-registration flow with post-registration enrichment plus source-local
concept-graph construction, while keeping the path offline-testable and visible
in the normal Layer 0 discovery / registration receipts.

## What Landed

- new offline-testable enrichment entrypoint:
  - `research/knowledge-hub/source-layer/scripts/enrich_with_deepxiv.py`
- new offline-testable concept-graph entrypoint plus schema:
  - `research/knowledge-hub/source-layer/scripts/build_concept_graph.py`
  - `research/knowledge-hub/schemas/concept-graph.schema.json`
- `register_arxiv_source.py` now supports integrated enrichment through:
  - `enrichment_override`
  - `skip_enrichment`
  - `--enrichment-json`
  - `--skip-enrichment`
- `register_arxiv_source.py` now also supports integrated graph construction through:
  - `graph_override`
  - `skip_graph_build`
  - `--graph-json`
  - `--skip-graph-build`
- `discover_and_register.py` now passes both enrichment and graph construction
  through the discovery bridge and records them in discovery receipts / JSON output
- the L0 discovery acceptance path now covers integrated enrichment instead of
  only raw registration:
  - `research/knowledge-hub/runtime/scripts/run_l0_source_discovery_acceptance.py`
- dedicated enrichment acceptance exists for the isolated register -> enrich
  path:
  - `research/knowledge-hub/runtime/scripts/run_l0_source_enrichment_acceptance.py`
- dedicated concept-graph acceptance exists for the isolated
  register -> enrich -> graph path:
  - `research/knowledge-hub/runtime/scripts/run_l0_source_concept_graph_acceptance.py`
- repository docs and attribution now surface the new Layer 0 artifacts:
  - `NOTICE`
  - `research/knowledge-hub/README.md`
  - `research/knowledge-hub/L0_SOURCE_LAYER.md`
  - `research/knowledge-hub/source-layer/README.md`
  - `research/knowledge-hub/runtime/README.md`
  - `research/knowledge-hub/runtime/AITP_TEST_RUNBOOK.md`

## Outcome

Most of `165.5-01` is now live.

- discovery -> registration no longer stops at metadata-only source capture
- enrichment receipts and DeepXiv-style provenance now exist on the default
  L0 registration path
- source-local concept graphs plus graph receipts now also exist on the default
  L0 registration path and are mirrored into the intake projection locator
- MIT attribution now exists at repository level via `NOTICE` and source-file headers
- the remaining open work inside `165.5-01` is still substantial:
  - richer Graphify-adapted extraction / dedup / caching than the current bounded offline contract
  - deeper graph semantics beyond the current minimal physics-oriented source artifact
  - the next major slice should move into `165.5-02`, where L1 distillation and intake start consuming the graph
