# Phase 165.5-03 Obsidian Export Summary

Status: partially implemented in working tree

## Goal

Open `165.5-03 / 999.84` by materializing an Obsidian-compatible concept-graph
export inside the existing `L1` vault, instead of leaving graph export as a
future external-only idea.

## What Landed

- new module:
  - `knowledge_hub/obsidian_graph_export.py`
- `l1_vault_support.py` now materializes a concept-graph export under:
  - `intake/topics/<topic_slug>/vault/wiki/concept-graph/`
- the export currently writes:
  - `manifest.json`
  - `index.md`
  - community-mapped folders of node notes
- node notes are Obsidian-compatible markdown with frontmatter and wikilinks
  for:
  - edge relations
  - hyperedge context
- source-local nodes are merged into a single exported note when they share:
  - normalized label
  - node type
- community labels now map directly to export folders
- the `L1` vault manifest now exposes the concept-graph export paths under the
  wiki payload, so the export is part of the durable vault contract rather than
  an out-of-band side effect
- isolated acceptance now exists for the end-to-end path:
  - `runtime/scripts/run_l1_graph_obsidian_export_acceptance.py`
