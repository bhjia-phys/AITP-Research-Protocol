# Phase 165.3-01 Summary

Status: implemented in working tree

## Goal

Open `165.3 / 999.60` by replacing the machine-ish default `status` / `next`
 CLI output with a short human-readable surface, while keeping the full
 dashboard available on demand.

## What Landed

- `aitp status` now defaults to a compact human summary instead of recursive
  dict dumping
- `aitp next` now defaults to a single actionable summary instead of a wall of
  raw payload keys
- new flag:
  - `aitp status --full`
- `status --full` now renders the persisted dashboard note directly when it is
  available, instead of forcing users to find the file path manually
- compact status rendering now includes:
  - topic
  - stage
  - mode
  - current bounded action
  - first mandatory read path
  - blocked status
  - machine-view command
- compact next rendering now includes:
  - topic
  - one explicit next action
  - first mandatory read path
  - blocked status
  - machine-view command
- implementation landed through:
  - `knowledge_hub/cli_compat_handler.py`
  - `knowledge_hub/aitp_cli.py`

`165.3` is now open as a real HCI phase:

  rather than a machine dump

- this gives the later onboarding and help-grouping work a cleaner base surface
