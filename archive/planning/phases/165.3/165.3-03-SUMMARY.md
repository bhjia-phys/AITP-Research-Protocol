# Phase 165.3-03 Summary

Status: implemented in working tree

## Goal

Open `165.3 / 999.63` by replacing the flat 60+ command cliff with a
progressive-disclosure CLI help surface:

- `aitp help` for the small operator-facing core
- `aitp help --all` for the full grouped inventory

## What Landed

- new human-oriented CLI entrypoint:
  - `aitp help`
- expanded grouped inventory surface:
  - `aitp help --all`
- `help` is now a real subcommand instead of forcing users into raw argparse
  output
- default `aitp help` is intentionally narrow and shows only the core 5
  commands required by the roadmap:
  - `session-start`
  - `status`
  - `next`
  - `work`
  - `consult-l2`
- `aitp help --all` now renders the full registered command set with grouped
  headings rather than one flat alphabetical wall
- grouping is generated from the parser's actual registered commands, so the
  expanded help surface stays aligned as new commands are added
- help rendering returns before service construction, keeping the path fast and
  side-effect free
- implementation landed through:
  - `knowledge_hub/cli_compat_handler.py`
  - `knowledge_hub/aitp_cli.py`
