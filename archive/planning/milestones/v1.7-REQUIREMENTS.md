# Requirements: v1.7 Kernel Decomposition And Maintainability Guard

**Defined:** 2026-04-05
**Status:** Shipped 2026-04-05
**Core Value:** AITP must turn vague but meaningful research starts into
bounded, durable, and explainable runtime state without pretending uncertainty
is already resolved.

## v1.7 Requirements

### Service And CLI Decomposition

- [x] **MTN-01**: Hotspot responsibilities in `aitp_service.py` have moved
  into focused modules without breaking the public API.
- [x] **MTN-02**: `aitp_cli.py` dispatch has begun moving into dedicated
  handler modules before more subcommands accumulate.

### Maintainability Guard

- [x] **MTN-03**: Maintainability budgets tightened as real extractions landed
  instead of staying frozen at old larger sizes.

### Docs And Integration

- [x] **DOC-06**: Docs and regression coverage now lock the new decomposition
  boundaries so future work does not silently re-centralize logic.

### Future Publication Layer

- [ ] **PUB-01**: `L5` should consume only completed topic outputs and relevant
  `L2` memory, not invent new scientific truth.
- [ ] **PUB-02**: `L5` should produce paper-grade structured outputs such as
  claim-evidence maps, outlines, figure plans, and manuscript drafts.
- [ ] **PUB-03**: `L5` should reject overclaiming or missing-evidence writing
  states by reopening topic work instead of patching them inside the writing
  layer.

## Future Requirements

- **SCRIPT-01**: Runtime script hotspots should later be decomposed or bounded
  with shared utility layers once service/CLI boundaries are clearer.
- **DIST-01**: Marketplace or one-click packaging can come after kernel
  maintainability is healthier.

## Out of Scope

| Feature | Reason |
|---------|--------|
| `L5` implementation in this milestone | `L5` remained a later final layer |
| New major protocol/runtime features | This milestone was about code structure, not feature expansion |
| Broad OpenClaw redesign | Not part of the immediate hotspot-decomposition target |
