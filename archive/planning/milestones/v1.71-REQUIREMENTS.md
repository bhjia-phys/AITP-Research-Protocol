# Requirements: v1.71 Runtime Transition And Demotion History

## Milestone Goal

Close the runtime-history gap so AITP can show how a topic moved across layers,
including demotions and backtracks, instead of only exposing current
`resume_stage` and `last_materialized_stage`.

## Completed Requirements

### Transition Trace

- [x] `REQ-TRANSITION-01`: the repository records one structured transition log
  for topic layer changes, including forward moves and backtracks, with
  explicit reason and evidence refs.
- [x] `REQ-TRANSITION-02`: the transition log is operator-visible through a
  runtime-facing read path rather than hidden only in implementation details or
  narrative logs.

### Demotion Honesty

- [x] `REQ-DEMOTION-01`: backward layer moves are first-class and keep their
  demotion reason visible instead of being flattened into a new current stage.
- [x] `REQ-DEMOTION-02`: promotion/demotion history stays compatible with the
  existing promotion-gate and promotion-trace surfaces rather than replacing
  them with a disconnected mechanism.

### Verification

- [x] `REQ-VERIFY-01`: the milestone closes with one bounded isolated
  transition/demotion acceptance path plus targeted regression coverage for the
  new history surfaces.

## Requirement Outcomes

- `REQ-TRANSITION-01`: validated through durable
  `transition_history.jsonl|json|md` surfaces fed by runtime state and
  promotion-gate events.
- `REQ-TRANSITION-02`: validated through runtime protocol note and topic replay
  exposure of transition-history paths and summaries.
- `REQ-DEMOTION-01`: validated through explicit `promotion_rejected` and
  `runtime_resume_state` backedge rows with recorded reasons.
- `REQ-DEMOTION-02`: validated through extension of the existing
  `promotion_gate` / `promotion_trace` ecosystem instead of a disconnected
  history subsystem.
- `REQ-VERIFY-01`: validated through the dedicated isolated transition-history
  acceptance lane and targeted regression slices.
