# Requirements: v1.6 Runtime Front-Door Parity Hardening

**Defined:** 2026-04-05
**Status:** Shipped 2026-04-05
**Core Value:** AITP must turn vague but meaningful research starts into
bounded, durable, and explainable runtime state without pretending uncertainty
is already resolved.

## v1.6 Requirements

### Runtime Support Matrix

- [x] **RTP-01**: `aitp doctor` now exposes a first-class runtime/front-door
  support matrix covering Codex, Claude Code, OpenCode, and OpenClaw.
- [x] **RTP-02**: The runtime matrix now distinguishes current baseline
  readiness, parity-hardening targets, and specialized lanes instead of
  implying uniform maturity.

### Claude And OpenCode Parity Hardening

- [x] **RTP-03**: Claude Code bootstrap health now exposes clear incomplete or
  stale-surface states rather than only a coarse missing-hook result.
- [x] **RTP-04**: OpenCode bootstrap health now exposes clear incomplete,
  compatibility, or stale-plugin states rather than only a coarse enabled
  flag.
- [x] **RTP-05**: Migration and install flows now make multi-runtime
  convergence expectations explicit so Codex success does not silently mask
  Claude/OpenCode drift.

### Docs And Integration

- [x] **DOC-05**: Docs and regression coverage now publish the current runtime
  support matrix honestly, keeping Codex as the present baseline and
  Claude/OpenCode as parity-hardening targets.

### Future Publication Layer

- [ ] **PUB-01**: `L5` should consume only completed topic outputs and relevant
  `L2` memory, not invent new scientific truth.
- [ ] **PUB-02**: `L5` should produce paper-grade structured outputs such as
  claim-evidence maps, outlines, figure plans, and manuscript drafts.
- [ ] **PUB-03**: `L5` should reject overclaiming or missing-evidence writing
  states by reopening topic work instead of patching them inside the writing
  layer.

## Future Requirements

- **AUTO-03**: Only after protocol-native reuse and compiled `L2` knowledge are
  proven should AITP consider adapter-facing bootstrap hints or generated skill
  wrappers.
- **GRAPH-01**: Cross-topic graph visualization may come later once compiled
  surfaces and hygiene semantics are stable.
- **DIST-01**: Marketplace or one-click packaging can come after runtime
  parity is honest and diagnostically inspectable.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Marketplace-packaging all runtimes | This milestone was about honest parity hardening, not final distribution polish |
| Replacing Codex as the default execution lane | The runtime still materially depends on Codex as the deepest baseline |
| Folding OpenClaw autonomy redesign into this milestone | OpenClaw is a specialized lane, not the target parity surface |
| Implementing `L5` in this milestone | `L5` is intentionally the later final output layer |
