# Requirements: v1.90 Hypothesis Route Transition Authority Surface

## Milestone Goal

Close the next post-`v1.89` research-control gap so AITP can turn transition
commitment into one operator-visible authority surface, instead of forcing
operators to infer whether a committed route has become the authoritative
bounded truth across runtime surfaces.

## Active Requirements

### Route Transition-Authority Surface

- [x] `REQ-HAUTH-01`: runtime surfaces can materialize a bounded route
  transition-authority summary from route transition commitment plus current
  route artifacts.
- [x] `REQ-HAUTH-02`: replay/read paths expose the authority decision directly
  instead of leaving that authority implicit.

### Coexistence Rule

- [x] `REQ-HAUTH-03`: the route transition-authority surface coexists cleanly
  with the current route_transition_commitment surface and helper mechanisms
  instead of auto-triggering fresh runtime mutation.

### Verification

- [x] `REQ-VERIFY-01`: the milestone closes with one bounded isolated
  route transition-authority acceptance path plus targeted regression coverage
  for the new authority surface.
