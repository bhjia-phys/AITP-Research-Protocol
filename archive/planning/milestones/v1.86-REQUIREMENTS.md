# Requirements: v1.86 Hypothesis Route Transition Clearance Surface

## Milestone Goal

Close the next post-`v1.85` research-control gap so AITP can turn transition
escalation into one operator-visible clearance surface, instead of forcing
operators to infer whether an escalated route transition is still blocked or
has been cleared after checkpoint lifecycle changes.

## Final Requirements Status

### Route Transition-Clearance Surface

- [x] `REQ-HCLEAR-01`: runtime surfaces can materialize a bounded route
  transition-clearance summary from route transition escalation plus
  operator-checkpoint lifecycle context.
- [x] `REQ-HCLEAR-02`: replay/read paths expose the clearance decision directly
  instead of leaving that clearance implicit.

### Coexistence Rule

- [x] `REQ-HCLEAR-03`: the route transition-clearance surface coexists cleanly
  with the current route_transition_escalation surface and helper mechanisms
  instead of auto-triggering fresh runtime mutation.

### Verification

- [x] `REQ-VERIFY-01`: the milestone closes with one bounded isolated
  route transition-clearance acceptance path plus targeted regression coverage
  for the new clearance surface.
