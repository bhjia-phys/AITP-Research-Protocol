# Roadmap: v1.44 L2 Knowledge-Network MVP Activation

## Overview

`v1.44` is shipped.

It activated the minimum useful `L2` knowledge-network surface in production
code: real MVP families, bounded seed/consult/compile/audit entrypoints, and
an isolated acceptance path proving one seeded direction end to end.

## Phases

- [x] **Phase 58: MVP Node-Family Activation And `physical_picture` Closure**
  - Goal: activate the bounded MVP `L2` family surface in schemas, docs, and
    seed/retrieval helpers, including `physical_picture`.
  - Outcome: the MVP subset is real production vocabulary rather than partly
    reserved design prose.

- [x] **Phase 59: Lightweight L2 Entry And Seed Command Family**
  - Goal: expose bounded production entrypoints for staging, seeding, and
    graph-facing inspection.
  - Outcome: L2 MVP work is reachable through service/CLI surfaces instead of
    living only in helper functions.

- [x] **Phase 60: Seeded Direction And Bounded Retrieval Proof**
  - Goal: seed one small real direction with nodes, edges, and consultable
    retrieval paths.
  - Outcome: `L2` is no longer just empty structure; one bounded direction is
    operational.

- [x] **Phase 61: L2 MVP Docs, Acceptance, And Regression Closure**
  - Goal: close the milestone with docs parity, bounded acceptance, and final
    regression closure.
  - Outcome: the L2 MVP surface can be resumed and audited from disk like the
    rest of AITP.

## Key Accomplishments

1. Activated `physical_picture` and the bounded MVP family surface across docs,
   schemas, backend targets, and seeded graph data.
2. Exposed public service/CLI entrypoints for seeding, consulting, compiling,
   and auditing bounded `L2` memory.
3. Closed the milestone with an isolated non-mocked MVP direction acceptance.
