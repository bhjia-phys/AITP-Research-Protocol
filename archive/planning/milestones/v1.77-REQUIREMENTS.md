# Requirements: v1.77 Hypothesis Route Handoff Surface

## Milestone Goal

Close the next post-`v1.76` research-control gap so AITP can turn ready
parked-route signals into an operator-visible route handoff surface, instead of
forcing operators to infer the next concrete route handoff from separate
activation, re-entry, and helper artifacts.

## Completed Requirements

### Route Handoff Surface

- [x] `REQ-HHANDOFF-01`: runtime surfaces can materialize a bounded handoff
  summary from current-route activation plus ready parked-route signals.
- [x] `REQ-HHANDOFF-02`: replay/read paths expose ready parked-route handoff
  candidates versus keep-parked decisions instead of leaving that handoff state
  implicit.

### Coexistence Rule

- [x] `REQ-HHANDOFF-03`: the route handoff surface coexists cleanly with the
  current route_activation surface, route_reentry surface, and helper
  mechanisms instead of auto-triggering runtime mutation.

### Verification

- [x] `REQ-VERIFY-01`: the milestone closes with one bounded isolated
  route handoff acceptance path plus targeted regression coverage for the new
  handoff surface.
