# Requirements: v1.92 Public Front Door Source Handoff

## Milestone Goal

Close the next post-`v1.91` usability gap by turning the honest post-bootstrap
return to `L0` into one concrete source-acquisition path and making the default
registration path contentful enough to be useful immediately.

## Archived Requirements

### Public Front Door Source Handoff

- [x] `REQ-L0HAND-01`: after a fresh public `bootstrap` returns a topic to
  `L0 source expansion`, the operator can see one concrete source-acquisition
  next step instead of only generic prose.
- [x] `REQ-L0HAND-02`: `status`, `runtime_protocol`, and `replay-topic` expose
  the same concrete `L0` handoff surface for the active topic.

### Contentful Registration Default

- [x] `REQ-L0HAND-03`: arXiv registration downloads source content by default,
  with `--metadata-only` as the explicit lightweight opt-out.

### Verification

- [x] `REQ-VERIFY-01`: the milestone closes with one bounded fresh-topic proof
  of `bootstrap -> concrete source handoff -> source registration` on a real
  repository kernel root.
