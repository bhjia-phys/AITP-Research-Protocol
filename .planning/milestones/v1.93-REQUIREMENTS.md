# Requirements: v1.93 L1 Contradiction Adjudication Surface

## Milestone Goal

Close the next post-`v1.92` intake-maturity gap by making `L1` surface
cross-source contradictions explicitly instead of leaving the operator to infer
them from separate assumption rows, reading-depth notes, and source summaries.

## Archived Requirements

### Contradiction Intake Surface

- [x] `REQ-L1CON-01`: `L1` intake records contradiction rows that make
  incompatible claims, assumptions, or regime statements explicit across
  multiple sources instead of leaving them as disconnected notes.
- [x] `REQ-L1CON-02`: each contradiction row carries source refs plus the
  bounded context needed to judge it, including claim summaries, reading-depth
  posture, and the comparison basis used to call it contradictory.
- [x] `REQ-L1CON-03`: `status`, `runtime_protocol`, and operator-facing `L1`
  views expose the same contradiction surface so the operator can act on it
  without opening raw intake JSON.

### Verification

- [x] `REQ-VERIFY-01`: the milestone closes with one bounded contradiction-aware
  multi-source intake proof that shows a fresh topic can surface a real
  contradiction record and replay it through the runtime read path.
