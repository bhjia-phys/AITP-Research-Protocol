# Requirements: v1.43 Unified Control Plane And Paired-Backend Closure

## Shipped Requirements

- ✓ `REQ-CTRL-01`: AITP exposes one explicit control-plane contract that keeps
  `task_type`, `lane`, `layer`, `mode`, and `H-plane` distinct in
  runtime/service/CLI surfaces.
- ✓ `REQ-CTRL-02`: Runtime truth and doctor/status surfaces explain the current
  control-plane state without forcing operators to reconstruct it from
  scattered docs or heuristics.
- ✓ `REQ-BACKEND-01`: Paired-backend alignment state and drift semantics are
  inspectable runtime and maintenance artifacts.
- ✓ `REQ-BACKEND-02`: Consultation, promotion, and paired-backend sync remain
  explicitly distinct operations with separate audit trails.
- ✓ `REQ-HPLANE-01`: Redirect, pause, approve, and override are materialized as
  durable `H-plane` artifacts and entrypoints.
- ✓ `REQ-HPLANE-02`: The runtime shell makes clear when human steering is
  blocking, guiding, or approving a control-plane transition.
- ✓ `REQ-VERIFY-01`: The milestone closes with a non-mocked verification path
  proving the new control-plane and paired-backend surfaces through production
  code.
